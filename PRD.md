# 微习惯 App 产品需求文档

## 1. 文档信息

产品名称：微习惯 App  
版本：MVP v0.1  
文档日期：2026-05-11  
目标平台：iOS 优先，后续适配 Android  
原型文件：[micro-habit-prototype.html](./micro-habit-prototype.html)

## 2. 产品定位

微习惯 App 是一款帮助用户跨过“开始阻力”的行为启动工具。

产品不以高压自律、惩罚断签、复杂目标管理为核心，而是通过“极小动作”和“1 分钟启动”帮助用户每天轻量开始，并在进入状态后顺势延展。

核心公式：

```text
微习惯 = 触发场景 + 最低完成 + 顺势继续 + 即时反馈 + 恢复记录
```

MVP 核心能力：

```text
动作启动：触发场景 → 完成一个极小动作 → 记录
计时启动：触发场景 → 1 分钟启动 → 5 分钟沉浸 → 记录
```

新增首版引导目标：

```text
让用户在首次进入时理解三件事：先开始 1 分钟、做一个小动作也算完成、暂停后可以接上。
```

## 3. 目标用户

### 3.1 核心用户

1. 想养成习惯但经常失败的人。
2. 容易被目标压力劝退的人。
3. 学生、职场人、自由职业者。
4. 有阅读、学习、写作、运动、冥想、整理等长期行为需求的人。

### 3.2 典型场景

用户想读书，但一想到“每天读 30 分钟”就抗拒。产品让用户只承诺“午饭后读 1 页”或“先开始 1 分钟”，完成后再邀请进入 5 分钟沉浸。

## 4. 用户痛点

1. 目标过大，开始前心理压力高。
2. 依赖意志力，没有稳定触发场景。
3. 普通打卡 App 重连续天数，断签后挫败感强。
4. 打卡记录成本高，用户完成动作后还要管理数据。
5. 不同行为类型需要不同启动方式，普通产品经常统一成一个“打卡”动作。

## 5. 产品目标

### 5.1 MVP 目标

验证用户是否愿意使用“双启动模式”完成微习惯闭环。

### 5.2 核心指标

1. 新用户首个习惯创建率：目标大于 70%。
2. 首日完成率：目标大于 60%。
3. D1 留存率：目标大于 35%。
4. D7 留存率：目标大于 18%。
5. 计时启动进入 5 分钟沉浸率：目标大于 30%。
6. 暂停后 2 天内恢复率：目标大于 40%。

### 5.3 反指标

1. 单次创建耗时超过 60 秒。
2. 用户创建超过 5 个活跃习惯。
3. 用户因断签提示产生负反馈。

## 6. 产品原则

1. 开始比完成大目标更重要。
2. 最低完成必须小到几乎不会失败。
3. 中断不是失败，恢复才是关键。
4. 完成反馈要即时，但不能打扰用户继续沉浸。
5. 创建流程优先推荐模板，减少用户思考。
6. MVP 不做社区、好友监督、复杂 AI 教练和会员体系。

## 7. 功能范围

### 7.1 MVP 包含

1. 今日页。
2. 首次使用引导。
3. 创建微习惯。
4. 动作启动模式。
5. 1 分钟启动模式。
6. 5 分钟沉浸模式。
7. 完成记录。
8. 暂停和恢复习惯。
9. 全部习惯记录总览。
10. 单个习惯记录详情。
11. 本地通知提醒。

### 7.2 MVP 不包含

1. 好友、群组、排行榜。
2. 付费会员。
3. 复杂数据图表。
4. AI 拆解目标。
5. 多端同步。
6. 课程内容。
7. 成就系统。

## 8. 信息架构

底部导航包含 3 个一级入口：

```text
今日
创建
记录
```

二级页面：

```text
1 分钟启动页
5 分钟沉浸页
单个习惯记录详情页
```

## 9. 核心用户流程

### 9.1 新用户创建第一个习惯

```text
打开 App
进入今日页空状态
点击创建
选择模板
选择启动方式
确认触发场景
生成微习惯
返回今日页
```

### 9.2 动作启动流程

```text
今日页
点击动作型习惯
用户完成动作
点击完成
出现即时反馈
进入该习惯详情
记录今日完成
```

### 9.3 计时启动流程

```text
今日页
点击计时型习惯
进入 1 分钟启动页
倒计时结束
提示进入 5 分钟沉浸
用户选择进入
完成 5 分钟或提前结束
进入该习惯详情
记录今日完成
```

### 9.4 查看记录流程

```text
底部导航点击记录
进入全部习惯记录总览
点击某个习惯
进入单个习惯详情
查看累计完成、恢复速度、本月日历
```

## 10. 页面需求

### 10.1 今日页

目标：让用户快速知道今天要启动什么，并用最低成本开始。

页面元素：

1. 今日日期。
2. 当前连续启动天数。
3. 推荐先启动的习惯。
4. 今日微习惯列表。
5. 每个习惯展示名称、触发场景、启动方式。
6. 计时启动按钮。
7. 动作完成按钮。
8. 创建入口。
9. 暂停中的习惯恢复入口。

交互规则：

1. 活跃习惯最多展示 3 个。
2. 今日页优先展示一个“推荐先启动”的习惯，降低用户决策成本。
3. 计时型习惯点击后进入 1 分钟启动页。
4. 动作型习惯点击完成后记录当前习惯。
5. 当天已完成的习惯保留在列表中，但状态显示为已完成。
6. 暂停习惯不进入今日待完成列表，但在恢复区展示。

### 10.2 创建页

目标：让用户在 60 秒内创建一个可执行微习惯。

字段：

1. 习惯目标：读书、写作、学习、运动、喝水、整理、冥想、睡眠。
2. 推荐方案：系统自动生成微动作、开始时机、启动方式和最低完成。
3. 微动作。
4. 什么时候开始。
5. 高级设置：启动方式、最低完成、顺势继续。
7. 频率，MVP 默认每天。

模板：

1. 读 1 页书。
2. 做 1 个深蹲。
3. 写 1 句话。
4. 背 1 个单词。
5. 冥想 1 分钟。

校验规则：

1. 微动作不能为空。
2. 开始时机不能为空。
3. 系统根据目标默认推荐启动方式，用户可在高级设置中修改。
4. 计时启动必须有启动时长，MVP 固定 1 分钟。
5. 活跃习惯超过 3 个时，提示用户先暂停一个已有习惯。

### 10.3 1 分钟启动页

目标：帮助用户进入行为状态。

页面元素：

1. 当前习惯名称。
2. 当前阶段：1 分钟启动。
3. 倒计时。
4. 暂停按钮。
5. 完成并记录。
6. 进入 5 分钟按钮。

规则：

1. 进入页面自动开始倒计时。
2. 倒计时结束后不强制进入 5 分钟，只给出邀请。
3. 用户完成 1 分钟即可算最低完成。
4. 用户退出时保留本次启动记录。
5. 所有文案必须明确：5 分钟是可选延展，不是完成条件。
6. 完成 1 分钟后进入完成反馈页，用户可选择回今日、顺势继续 5 分钟或查看记录。

### 10.4 5 分钟沉浸页

目标：让已经开始的用户自然延展行为。

页面元素：

1. 当前阶段：5 分钟沉浸。
2. 倒计时。
3. 完成并记录。
4. 再来 5 分钟。
5. 退出并记录。

规则：

1. 5 分钟不是强制要求。
2. 提前退出也算已完成最低启动。
3. 再来 5 分钟可重复触发，MVP 不限制次数。
4. 页面不得制造“没做 5 分钟就失败”的感受。

### 10.5 全部记录总览页

目标：从全局视角查看所有习惯。

页面元素：

1. 今日完成数。
2. 本周启动次数。
3. 今天建议接上的习惯。
4. 习惯概览列表。
5. 每个习惯展示累计完成、本周次数、进度条。

规则：

1. 底部导航“记录”默认进入全部记录总览。
2. 点击某个习惯进入单习惯详情。

### 10.6 单个习惯详情页

目标：查看一个习惯自己的完成情况。

页面元素：

1. 习惯名称。
2. 触发场景。
3. 累计完成。
4. 恢复速度。
5. 本月日历。
6. 恢复提示文案。

规则：

1. 从具体习惯完成后进入对应习惯详情。
2. 详情页不得混用其他习惯数据。
3. 日历显示完成、暂停、未开始三种状态。

## 11. 数据定义

### 11.1 习惯 Habit

| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 习惯 ID |
| title | string | 微动作名称 |
| trigger | string | 触发场景 |
| startMode | enum | action 或 timer |
| minimumTarget | string | 最低完成 |
| extensionTarget | string | 顺势继续 |
| frequency | enum | MVP 固定 daily |
| status | enum | active、paused、archived |
| createdAt | datetime | 创建时间 |
| updatedAt | datetime | 更新时间 |

### 11.2 完成记录 HabitLog

| 字段 | 类型 | 说明 |
|---|---|---|
| id | string | 记录 ID |
| habitId | string | 习惯 ID |
| date | date | 归属日期 |
| status | enum | completed、paused、skipped |
| source | enum | action、timer_1m、focus_5m |
| durationSeconds | number | 本次计时时长 |
| focusRounds | number | 5 分钟沉浸轮数 |
| createdAt | datetime | 创建时间 |

## 12. 统计口径

1. 今日完成数：当天 status 为 completed 的不同 habitId 数量。
2. 累计完成：某个 habitId 下 completed 记录数量。
3. 本周启动：本周 completed 记录数量，包含 action、timer_1m、focus_5m。
4. 连续启动天数：连续存在 completed 记录的自然日数量。
5. 恢复速度：某习惯最近一次暂停或未完成后，到下一次 completed 的间隔天数。
6. 5 分钟沉浸率：进入 5 分钟沉浸的人数 / 完成 1 分钟启动的人数。

## 14. 通知需求

MVP 支持本地通知，不做服务端推送。

规则：

1. 用户创建习惯时可选择提醒时间。
2. 默认不强制开启提醒。
3. 通知文案避免压力化表达。

文案示例：

```text
现在只需要开始 1 分钟
今天做一个小动作就够了
接上今天，不用补偿昨天
```

## 15. 异常与边界

1. 用户当天重复完成同一习惯：更新当天记录，不重复计入今日完成数。
2. 计时中切到后台：记录当前剩余时间；返回后继续或手动完成。
3. 用户删除习惯：MVP 改为归档，不物理删除历史记录。
4. 用户跨时区：MVP 按设备本地日期归属。
5. 用户无网络：MVP 本地可用，不依赖网络。

## 16. 验收标准

1. 用户可在 60 秒内创建一个微习惯。
2. 今日页能正确区分动作启动和计时启动。
3. 动作启动完成后进入对应习惯详情。
4. 计时启动完成后进入对应习惯详情。
5. 底部“记录”进入全部习惯总览，不是单习惯详情。
6. 单习惯详情只展示当前习惯的数据。
7. 1 分钟和 5 分钟倒计时可开始、暂停、完成。
8. 本地刷新或重新打开后，已有习惯和记录不丢失。
